# EDU Suite — Use Case Table

> Following Raffaele's structure: Use Case → Target → Trigger → Steps → Results → Success Criteria

## Use Cases

| # | Use Case | 6C Category | Persona | Trigger | Steps | Expected Output (JSON) | UI Component | Success Criteria |
|---|----------|-------------|---------|---------|-------|----------------------|--------------|------------------|
| 1 | **Generate Quiz from Content** | Engagement | Student / Teacher | User pastes text or uploads document | 1. Parse content 2. Identify key concepts 3. Generate MCQ + open-ended 4. Create answer key | `QuizSchema` — questions[], correctAnswers[], difficulty | QuizRunner | Quiz covers main topics; 90%+ questions answerable from source; consistent format across runs |
| 2 | **Create Flashcard Deck** | Conversion | Student | User provides notes or text | 1. Extract key terms 2. Generate front/back pairs 3. Assign difficulty levels | `FlashcardSchema` — cards[]{front, back, difficulty} | FlashcardDeck | Cards cover core concepts; difficulty progression makes sense; no duplicate cards |
| 3 | **Build Study Plan** | Creation | Student | User provides syllabus + exam date | 1. Parse topics 2. Estimate time per topic 3. Schedule sessions 4. Add milestones | `StudyPlanSchema` — sessions[], milestones[], totalDuration | StudyPlanView | Plan is realistic; covers all topics; sessions fit available time |
| 4 | **Create Study Package from Files** | Connection | Student / Teacher | User provides file path or uploads docs | 1. Read file(s) 2. Extract content 3. Identify key concepts 4. Structure as package | `StudyPackageSchema` — title, keyConcepts[], sections[], readingOrder | StudyPackageCard | Package summarizes content accurately; key concepts are relevant |
| 5 | **Generate Lesson Plan** | Creation | Teacher | Teacher specifies topic + grade + duration | 1. Define objectives 2. Plan activities 3. Add assessments 4. Include resources | `LessonPlanSchema` — objectives[], activities[], assessments[], duration | LessonPlanView | Aligns with curriculum standards; activities match grade level |
| 6 | **Create Rubric** | Creation | Teacher | Teacher describes assignment | 1. Identify criteria 2. Define performance levels 3. Add descriptions 4. Assign weights | `RubricSchema` — criteria[], levels[], weights | RubricTable | Criteria are relevant; levels are distinct; total weight = 100% |
| 7 | **Summarize Content at Grade Level** | Knowledge | Student / Teacher | User provides text + target grade | 1. Analyze complexity 2. Simplify language 3. Extract key points 4. Structure summary | `SummarySchema` — summary, keyPoints[], gradeLevel, readingTime | SummaryCard | Reading level matches target; key info preserved |
| 8 | **Generate Practice Problems** | Engagement | Student | User specifies topic + difficulty | 1. Generate problems 2. Create solutions 3. Add hints 4. Grade by difficulty | `PracticeProblemsSchema` — problems[]{question, hint, solution, difficulty} | ProblemSet | Problems are solvable; difficulty is appropriate; solutions are correct |
| 9 | **Build Vocabulary List** | Conversion | Student | User provides text | 1. Extract key terms 2. Generate definitions 3. Add examples 4. Create usage sentences | `VocabularySchema` — terms[]{term, definition, example, partOfSpeech} | VocabularyList | Terms are relevant; definitions are accurate; examples are helpful |
| 10 | **Generate Concept Map** | Knowledge | Student / Teacher | User provides topic or text | 1. Identify concepts 2. Map relationships 3. Create hierarchy 4. Add descriptions | `ConceptMapSchema` — nodes[]{id, label}, edges[]{from, to, relationship} | ConceptMapView | Concepts are accurate; relationships make sense; hierarchy is logical |
| 11 | **Import from Google Drive** | Connection | Teacher / Admin | User provides Drive folder URL | 1. Auth with Google 2. List files 3. Extract content 4. Create study materials | Depends on downstream use case | FileImporter | Files retrieved successfully; content parsed correctly |
| 12 | **Essay Feedback** | Knowledge | Teacher | Teacher provides essay + rubric | 1. Analyze against rubric 2. Score each criterion 3. Generate feedback 4. Suggest improvements | `FeedbackSchema` — scores[], feedback[], suggestions[], overallGrade | FeedbackReport | Feedback is constructive; scores align with rubric; suggestions are actionable |
| 13 | **Schedule Study Sessions** | Connection | Student | User provides plan + calendar access | 1. Read study plan 2. Check calendar availability 3. Create calendar events 4. Set reminders | `CalendarEventsSchema` — events[]{title, start, end, reminder} | CalendarView | Events don't conflict; reminders are set; all topics covered |
| 14 | **Content Moderation Check** | Control | Admin | Admin submits content for review | 1. Scan for inappropriate content 2. Check age-appropriateness 3. Flag issues 4. Generate report | `ModerationSchema` — flags[], severity, recommendations | ModerationReport | Catches inappropriate content; false positive rate < 5% |
| 15 | **Generate Assessment from Learning Objectives** | Creation | Teacher | Teacher provides learning objectives | 1. Parse objectives 2. Map to Bloom's taxonomy 3. Generate questions per level 4. Create rubric | `AssessmentSchema` — questions[], bloomLevel[], rubric | AssessmentBuilder | Questions match objectives; all Bloom's levels represented |
